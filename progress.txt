- Batch INSERT via executemany in single transaction per table

## Story #3 - Removal Detection (2026-03-19)
- Removal detection queries DB for all active keys (grouped by natural key, _removed_date IS NULL/empty)
- Compares against source_keys (key_occurrence dict from insert phase) to find missing keys
- Updates only the latest row (MAX(rowid)) per missing key with _removed_date = today
- Keys that reappear later get a new INSERT naturally (hash-dedup treats them as new)
- Confirmed 0 removals on unchanged source data across all 8 tables
